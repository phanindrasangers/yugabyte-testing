# grafana-dashboards

A generic dashboards-as-code Helm chart. It onboards Grafana dashboards for any
component in the cluster, organised into folders, by turning each dashboard JSON into
a ConfigMap labelled for the Grafana sidecar. It does not install Grafana; it feeds an
existing one (for example the Grafana in kube-prometheus-stack) through its dashboard
sidecar.

## The idea

Folders on disk become folders in Grafana:

```
files/dashboards/
  YugabyteDB/
    yb-fleet-overview.json
    yb-env-detail.json
  Platform/
    prometheus-targets.json
  Kubernetes/            # add any component by adding a folder
    ...
  PostgreSQL/
    ...
```

Drop a JSON under `files/dashboards/<Folder>/` and run `helm upgrade`. The subdirectory
name is the Grafana folder. Files placed directly under `files/dashboards/` land in
`defaultFolder` (General by default). No template edits, ever.

## Install

```bash
helm install grafana-dashboards monitoring/charts/grafana-dashboards -n monitoring
```

## Values

| Key | Default | Description |
|-----|---------|-------------|
| `sidecar.label` | `grafana_dashboard` | Label the Grafana sidecar watches. |
| `sidecar.labelValue` | `"1"` | Value for that label. |
| `folderAnnotation` | `grafana_folder` | Annotation the sidecar reads for the folder. |
| `defaultFolder` | `General` | Folder for dashboards placed at `files/dashboards/` root. |
| `namespaceOverride` | `""` | ConfigMap namespace. Empty means release namespace. |
| `datasource.uid` | `""` | If set, replaces `${datasource}` in every dashboard. Pin globally. |
| `commonLabels` / `commonAnnotations` | `{}` | Added to every ConfigMap. |
| `folders.<Folder>.enabled` | `true` | Enable/disable a whole component folder. |
| `folders.<Folder>.folder` | the key | Display-name override for the folder. |
| `folders.<Folder>.datasourceUid` | `datasource.uid` | Pin a data source for one folder. |
| `folders.<Folder>.dashboards.<name>.enabled` | `true` | Toggle a single dashboard. |
| `extraDashboards` | `[]` | Inline dashboards supplied through values (name, folder, json). |

## Examples

Disable a whole component and pin one folder to a specific data source:

```bash
helm upgrade grafana-dashboards monitoring/charts/grafana-dashboards -n monitoring \
  --set folders.Kubernetes.enabled=false \
  --set folders.YugabyteDB.datasourceUid=prometheus
```

Disable a single dashboard:

```bash
--set folders.YugabyteDB.dashboards.yb-env-detail.enabled=false
```

Add a community dashboard inline without committing a file:

```yaml
extraDashboards:
  - name: node-exporter-full
    folder: Kubernetes
    json: |-
      { "title": "Node Exporter Full", "panels": [ ... ] }
```

## Where the JSON comes from

The YugabyteDB and Platform dashboards are generated by
`monitoring/generate_dashboards.py`, which writes straight into
`files/dashboards/<Folder>/`. For other components, vendor the JSON from the source
(Grafana export, community dashboard, your own generator) into the matching folder.

## Requirements

Grafana must run the dashboard sidecar with `searchNamespace: ALL` (or this chart's
namespace) and a matching `sidecar.label`. kube-prometheus-stack satisfies this when
`grafana.sidecar.dashboards.enabled=true`.
